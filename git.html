Rebasing (Advanced)

What is Rebase?

Rebase moves your commits to a new base commit, creating a clean linear history

$ git checkout feature/new-feature
$ git rebase main

Rebase Transformation

BEFORE Rebase: 

main:     A---B---C---D
               \
feature:        E---F  ‚Üê original commits
‚¨á git rebase main ‚¨á

AFTER Rebase: 

main:     A---B---C---D
                           \
feature:                E'---F'  ‚Üê NEW commits (same code, different IDs)

Rebase vs Merge - When to Use?

Use Merge When:

  • Working on a shared branch
  • You want to preserve history
  • Collaborating with others

Use Rebase When:

  • Cleaning up local commits
  • Creating linear history
  • Before pushing to remote

Critical Rule: Don't Rebase Shared Code!

‚úÖ SAFE: Rebase commits that are only on your local machine
‚ùå DANGER: Rebase commits that you've already pushed where others can see them

Why? Rebase creates new commits with different IDs.
This breaks the history for teammates working on the same branch!

Handling Merge Conflicts

When you see a conflict:

CONFLICT (content): Merge conflict in index.html

Open the file, you'll see:

<<<<<<< HEAD
Your changes
=======
Their changes
>>>>>>> feature-branch
1. Edit the file, choose what to keep
2. Remove conflict markers (<<<<, ====, >>>>)
3. Stage the resolved file: git add index.html
4. Complete the merge: git commit -m "Resolve merge conflict"

11 Best Practices

Commit Often

  • Make small, focused commits
  • Push at least once a day
  • Don't let local branch get too far ahead

Meaningful Messages

  • Be clear and descriptive
  • Explain WHY, not just WHAT
  • Follow team conventions

Use Branches

  • Never work directly on main
  • Create feature branches
  • Delete merged branches

Pull Before Push

  • Always pull latest changes first
  • Prevents conflicts
  • Keeps code up to date

Troubleshooting Common Issues

"fatal: not a git repository"

# You're not in a Git repo. Initialize one:
$ git init

"remote origin already exists"

# Remove existing remote first:
$ git remote remove origin
$ git remote add origin <new-url>

"Updates were rejected"

# Pull first, then push:
$ git pull origin main
$ git push origin main
