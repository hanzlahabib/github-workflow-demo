🌿 Branch & Commit Standards

🌿 Branch Naming

Format:

<issue-id>-<short-description>

Examples:

890-fix-api 12-fix-button 567-resolve-crash

Rules:
✓ Lowercase only
✓ Use hyphens (-) as separators
✓ Keep it short (3-5 words)

💬 Commit Messages

Pattern:

<type>(scope): <summary>

Examples:

feat(web): add OAuth2 support fix(server): resolve null pointer docs(ci): update deployment guide

Types:
feat, fix, docs, style,
refactor, test, chore,
perf, revert

Scopes:
web, server, docker,
k8s, e2e, ci, deps,
ml, keycloak

❌ Complex Templates

Keep it simple initially

❌ Forcing Everything

Allow flexibility for exceptions

❌ No Training

Invest in team education

❌ No Metrics

Measure from day 1
